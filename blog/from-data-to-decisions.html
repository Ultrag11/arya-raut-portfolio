Personal Essay

From Data to Decisions

I like data most when it helps a team decide what to do next.

Tools like Python, SQL, Minitab, Tableau, Power BI, Excel Solver, and R are useful because they can reveal patterns that are hard to see during daily operations. But data by itself does not improve a process. The value comes when analysis connects to a decision: which station needs attention, which sequence reduces idle time, which factor affects downtime, or which improvement should be tested first.

My downtime optimization project made that clear. A mixed-integer linear programming model is technical, but the purpose was practical: reduce idle time, improve throughput, and understand cost sensitivity across a production system. The model mattered because it turned a complicated scheduling problem into options that could be compared.

I see analytics as a bridge between engineering judgment and operational action. It can support a Kaizen event, validate a quality improvement, size the impact of a fixture change, or help communicate why a recommendation is worth doing.

That is the balance I want in my work: technical enough to be rigorous, clear enough to be useful, and grounded enough to improve the real system.
